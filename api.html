API Documentation

GET

/api/listfiles
Outputs the names of the files directly inside of /projects, seperated by line breaks(\n)
/api/listthemes
Outputs a list of theme names in the same format as listfiles
/api/readfile
file
: Name of the file in /projects to read
Outputs the content of the file denoted by the file arguement
/api/hostname
Outputs the Pi's hostname

POST

/api/savefile
file
: Name of the file in /projects to read
content
: The data to write to the file
Outputs the content of the file denoted by the file arguement

Web Socket

/api/socket
The server expects to receive the name of the file(Only the name, /projects/ is prefixed to it by the server) to run as the first message
Once the server has received the filename, it will create a subprocess and start sending output back prefixed with "output: " any server error messages will be preceeded by "error: "
To simulate a Control+C message, send "close", the process may not halt, since the python script could handle the KeyboardInterrupt
The server will close the socket when the process has halted
